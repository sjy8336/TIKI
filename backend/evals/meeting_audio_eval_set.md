# Meeting Audio Evaluation Set

이 문서는 STT/요약 품질을 확인하기 위한 실제 회의 음성 기준선이다.
같은 파일을 다시 돌렸을 때 결과가 얼마나 흔들리는지 확인하는 회귀용 메모로 사용한다.

## 실행 기준

- 현재 파이프라인은 `WHISPER_MODEL=small`, `WHISPER_NOISY_MODEL=medium` 구성을 쓰되, noisy 파일은 캐시가 있을 때만 noisy 모델로 올라간다.
- 현재 파이프라인은 `WHISPER_MODEL=large`, `WHISPER_LIGHT_MODEL=small` 하이브리드 구성을 쓴다.
- OpenAI 분석 경로가 실패하면 heuristic fallback으로 결과를 만든다.
- noisy 회의는 fixed window split + overlap 전략을 사용한다.
- `STT_ROUTING` 로그를 통해 chunk별 모델 선택을 같이 확인한다.
- 재실행은 `python scripts/analyze_meeting_audio.py "<audio-path>" --mode heuristic`로 확인한다.

## 관찰 포인트

- 긴 회의는 chunking이 잘 걸리는지 먼저 본다.
- noisy 파일에서 경계 중복이 과하지 않은지 확인한다.
- `결정`, `액션`, `리스크`, `다음 회의`가 최소 2개 이상 잡히는지 확인한다.
- 같은 파일을 다시 돌렸을 때 출력이 크게 흔들리지 않는지 비교한다.

## 다음 비교 기준

- `회의1.m4a`는 실제 업무관리시스템 회의라서, 액션 아이템과 일정이 핵심이다.
- `20260617_그럼_다들_모였으니.mp3`는 개발/버그 수정과 후속 확인이 핵심이다.

## 현재 한계

- OpenAI 경로는 현재 환경에서 연결 실패가 나서 heuristic fallback 기준으로 기록했다.
- 일부 STT 오인식은 남아 있다. 이 문서는 절대 정답이 아니라 회귀 감지용 baseline이다.
